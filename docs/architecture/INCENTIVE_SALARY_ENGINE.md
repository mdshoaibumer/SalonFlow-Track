# Incentive & Salary Engine Design

## Overview

The incentive engine is a **rule-based commission calculation system** that supports
multiple incentive strategies, applied per-staff or globally, with configurable slabs.

---

## Incentive Rule Types

### 1. Revenue Slab (`revenue_slab`)

Commission based on total revenue generated by a staff member in a period.

```
Example: Monthly Revenue Commission
┌────────────────────────────────┬───────────────┐
│ Revenue Range                  │ Commission    │
├────────────────────────────────┼───────────────┤
│ ₹0 - ₹10,000                  │ 0%            │
│ ₹10,001 - ₹20,000             │ 5%            │
│ ₹20,001 - ₹35,000             │ 8%            │
│ ₹35,001+                      │ 10%           │
└────────────────────────────────┴───────────────┘

Staff: Nazim
Monthly Revenue: ₹25,000
Slab: ₹20,001 - ₹35,000 → 8%
Commission: ₹25,000 × 8% = ₹2,000
```

**Evaluation**: The ENTIRE revenue falls into ONE slab (highest matching).
This is the "whole amount at slab rate" model (not progressive).

### 2. Service-Based (`service_based`)

Commission per specific service or service category performed.

```
Example: Hair Color Commission
Service: Hair Color
Commission: 15% of service price

Staff: Nazim did 5 Hair Colors @ ₹2,000 each
Revenue from Hair Color: ₹10,000
Commission: ₹10,000 × 15% = ₹1,500
```

### 3. Product Sale (`product_sale`)

Commission on products sold to customers.

```
Example: Product Sales Commission
Commission: 10% of product selling price

Staff: Sold shampoo ₹500 + serum ₹800
Revenue: ₹1,300
Commission: ₹1,300 × 10% = ₹130
```

```
├─────────────────────────────────────────────────────┤
│                                                       │
│  1. Identify period (month start → month end)        │
│                                                       │
│  2. For each active staff member:                    │
│     a. Get all active rules (global + staff-specific)│

│  3. StaffIncentive status = "calculated"             │
│                                                       │

│                                                       │
│  5. Approved incentives included in salary           │
│                                                       │
└─────────────────────────────────────────────────────┘
```

## Revenue Slab Calculation Algorithm

```go
func CalculateRevenueSlab(revenue float64, slabs []Slab) float64 {
    // Slabs are sorted by min_amount ASC
    // Find the highest slab where revenue >= min_amount
    var matchedSlab *Slab
    for i := range slabs {
        if revenue >= slabs[i].MinAmount {
            if slabs[i].MaxAmount == nil || revenue < *slabs[i].MaxAmount {
                matchedSlab = &slabs[i]
                break // First match in sorted order
            }
        }
    }
    
    if matchedSlab == nil {
        return 0
    }
    
    switch matchedSlab.CommissionType {
    case "percentage":
        return revenue * (matchedSlab.CommissionValue / 100)
    case "fixed":
        return matchedSlab.CommissionValue
    }
    return 0
}
```

## Service-Based Calculation Algorithm

```go
func CalculateServiceBased(rule IncentiveRule, staffID uuid.UUID, periodStart, periodEnd time.Time) float64 {
    // Query: sum of invoice_items.total_price
    //   WHERE staff_id = staffID
    //   AND service_id = rule.ServiceID
    //   AND invoice.date BETWEEN periodStart AND periodEnd
    //   AND invoice.status = 'completed'
    serviceRevenue := queryServiceRevenue(staffID, rule.ServiceID, periodStart, periodEnd)
    
    // Apply first slab (service rules typically have one slab)
    slab := rule.Slabs[0]
    switch slab.CommissionType {
    case "percentage":
        return serviceRevenue * (slab.CommissionValue / 100)
    case "fixed":
        return slab.CommissionValue * countServices // fixed per service
    }
    return 0
}
```

## Salary Generation Flow

```
┌─────────────────────────────────────────────────────────┐
│              SALARY GENERATION PIPELINE                    │
├─────────────────────────────────────────────────────────┤
│                                                           │
│  Input: staff_id, month, year                            │
│                                                           │
│  Step 1: Get staff.base_salary                           │
│          → Create SalaryLineItem (type: base_pay)        │
│                                                           │
│  Step 2: Get approved staff_incentives for the month     │
│          → For each incentive:                           │
│            Create SalaryLineItem (type: commission)      │
│            Set reference_id = staff_incentive.id         │
│                                                           │
│  Step 3: Get pending/partial advances                    │
│          → For each advance:                             │
│            Create SalaryLineItem (type: advance_deduction)│
│            Set reference_id = advance.id                 │
│            Call advance.Recover(amount)                   │
│                                                           │
│  Step 4: Apply any manual adjustments                    │
│          → bonus, other_deduction, other_earning         │
│                                                           │
│  Step 5: Recalculate totals                              │
│          total_earnings = base + Σ(positive items)       │
│          total_deductions = Σ(negative items)            │
│          net_salary = total_earnings - total_deductions  │
│                                                           │
│  Step 6: Save Salary with status = "draft"               │
│                                                           │
│  Step 7: Owner reviews → Approve → Pay                   │
│                                                           │
└─────────────────────────────────────────────────────────┘
```

## Rule Precedence

When multiple rules apply to the same staff member:

1. **Staff-specific rules** override **global rules** of the same type
2. **Service-based** and **revenue-slab** rules are **additive** (both apply)
3. Only rules within their `effective_from` → `effective_to` date range apply
4. Only `is_active = true` rules are evaluated

## Edge Cases

| Scenario | Handling |
|----------|----------|
| Staff has no base salary | Use 0, only commission applies |
| No incentive rules defined | Salary = base salary only |
| Advance exceeds net salary | Carry forward to next month |
| Mid-month staff join | Pro-rate base salary |
| Mid-month rule change | Use rule active on last day of month |
| Staff on leave (no revenue) | Salary = base only, no commission |
